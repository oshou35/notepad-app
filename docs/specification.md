# Notepad App 仕様書

## 1. アプリケーション概要

Notepad Appは、シンプルで使いやすいメモ帳アプリケーションです。ブラウザ上で動作し、LocalStorageを使用してメモデータを永続化します。

### 1.1 目的
- ユーザーが簡単にメモを作成・編集・削除できる
- データを自動的に保存し、ブラウザを閉じてもデータが保持される
- 直感的で使いやすいUIを提供する

### 1.2 対象ユーザー
- 日常的にメモを取る必要があるユーザー
- シンプルなメモ帳機能を求めるユーザー

## 2. 機能仕様

### 2.1 メモ作成機能
- **機能名**: 新規メモ作成
- **説明**: ユーザーが新しいメモを作成できる
- **動作**:
  - 「+ 新規メモ」ボタンをクリックすると、新しいメモが作成される
  - 作成されたメモは自動的に選択状態になる
  - メモIDは現在のタイムスタンプ（`Date.now()`）を文字列化したものを使用
  - 初期状態ではタイトルと本文は空文字列
  - 作成日時（`createdAt`）と更新日時（`updatedAt`）は現在時刻が設定される
  - 新しく作成されたメモは一覧の先頭に表示される

### 2.2 メモ編集機能
- **機能名**: メモ編集
- **説明**: 既存のメモのタイトルと本文を編集できる
- **動作**:
  - メモ一覧からメモを選択すると、右側のエディターに表示される
  - タイトル入力欄と本文入力欄が表示される
  - タイトルまたは本文を変更すると、自動的に`updatedAt`が更新される
  - 編集内容は即座にLocalStorageに保存される（自動保存）

### 2.3 メモ削除機能
- **機能名**: メモ削除
- **説明**: 不要になったメモを削除できる
- **動作**:
  - 各メモ項目の右上に「×」ボタンが表示される
  - 「×」ボタンをクリックすると、そのメモが削除される
  - 削除されたメモが現在選択されている場合、選択状態が解除される
  - 削除操作は即座にLocalStorageに反映される

### 2.4 メモ一覧表示機能
- **機能名**: メモ一覧表示
- **説明**: すべてのメモを一覧で表示する
- **動作**:
  - 左側のパネルにメモ一覧が表示される
  - メモは更新日時の新しい順に自動的にソートされる
  - 各メモ項目には以下が表示される:
    - タイトル（空の場合は「無題のメモ」と表示）
    - 本文のプレビュー（最初の50文字）
    - 更新日時（日本語形式で表示: 例「1月15日 14:30」）
  - 選択されているメモは視覚的にハイライト表示される
  - メモが存在しない場合は「メモがありません」と表示される

### 2.5 データ永続化機能
- **機能名**: LocalStorage自動保存
- **説明**: メモデータをブラウザのLocalStorageに自動保存する
- **動作**:
  - メモの作成・編集・削除が行われると、即座にLocalStorageに保存される
  - アプリ起動時、LocalStorageからデータを読み込む
  - LocalStorageのキー名は`'notes'`
  - データはJSON形式で保存される
  - LocalStorageへのアクセスに失敗した場合、エラーがコンソールに出力されるが、アプリは動作を継続する

## 3. UI/UX仕様

### 3.1 レイアウト
- **構成**: 2カラムレイアウト
  - 左側: メモ一覧パネル（`NoteList`コンポーネント）
  - 右側: メモエディターパネル（`NoteEditor`コンポーネント）

### 3.2 メモ一覧パネル
- **ヘッダー**:
  - 「メモ一覧」という見出し
  - 「+ 新規メモ」ボタン
- **メモ項目**:
  - クリック可能なカード形式
  - 選択状態のメモは視覚的に区別される（`selected`クラス）
  - 各項目に削除ボタン（×）が表示される

### 3.3 メモエディターパネル
- **空状態**:
  - メモが選択されていない場合、「メモを選択するか、新しいメモを作成してください」というメッセージが表示される
- **編集状態**:
  - タイトル入力欄（1行テキスト入力）
  - 本文入力欄（複数行テキストエリア）
  - プレースホルダーテキストが表示される

### 3.4 インタラクション
- **メモ選択**: メモ一覧の項目をクリックすると、そのメモが選択され、エディターに表示される
- **削除操作**: 削除ボタンをクリックすると、イベントの伝播が停止され、メモが削除される（メモ選択イベントは発生しない）

## 4. データ構造

### 4.1 Note型定義
```typescript
interface Note {
  id: string;          // 一意の識別子（タイムスタンプの文字列）
  title: string;       // メモのタイトル
  content: string;     // メモの本文
  createdAt: number;   // 作成日時（Unixタイムスタンプ）
  updatedAt: number;   // 更新日時（Unixタイムスタンプ）
}
```

### 4.2 LocalStorage構造
- **キー**: `'notes'`
- **値**: `Note[]`型の配列をJSON文字列化したもの
- **例**:
```json
[
  {
    "id": "1705123456789",
    "title": "サンプルメモ",
    "content": "これはサンプルメモです",
    "createdAt": 1705123456789,
    "updatedAt": 1705123456789
  }
]
```

## 5. 技術仕様

### 5.1 技術スタック
- **React** 19.1.1: UIライブラリ
- **TypeScript** 5.9.3: 型安全性の確保
- **Vite** 7.1.7: ビルドツールと開発サーバー
- **Playwright** 1.55.1: E2Eテストフレームワーク

### 5.2 コンポーネント構成

#### 5.2.1 App.tsx（メインコンポーネント）
- **役割**: アプリケーションの状態管理と全体の制御
- **状態管理**:
  - `notes`: メモの配列（LocalStorageと同期）
  - `selectedNoteId`: 現在選択されているメモのID
- **主要関数**:
  - `createNote()`: 新規メモ作成
  - `updateNote(id, updates)`: メモ更新
  - `deleteNote(id)`: メモ削除
- **ソート処理**: メモを`updatedAt`の降順でソート

#### 5.2.2 NoteList.tsx（メモ一覧コンポーネント）
- **役割**: メモ一覧の表示と操作
- **Props**:
  - `notes`: 表示するメモの配列
  - `selectedNoteId`: 選択されているメモのID
  - `onSelectNote`: メモ選択時のコールバック
  - `onCreateNote`: 新規メモ作成時のコールバック
  - `onDeleteNote`: メモ削除時のコールバック
- **機能**:
  - 日時フォーマット（日本語形式）
  - メモプレビュー表示（最初の50文字）

#### 5.2.3 NoteEditor.tsx（メモエディターコンポーネント）
- **役割**: メモの編集
- **Props**:
  - `note`: 編集対象のメモ（nullの場合は空状態）
  - `onUpdateNote`: メモ更新時のコールバック
- **機能**:
  - タイトル入力
  - 本文入力
  - 更新日時の自動更新

### 5.3 カスタムフック

#### 5.3.1 useLocalStorage.ts
- **役割**: LocalStorageとの同期を管理するカスタムフック
- **機能**:
  - 初期値の読み込み（LocalStorageから）
  - 値の更新とLocalStorageへの保存
  - エラーハンドリング
- **使用例**:
```typescript
const [notes, setNotes] = useLocalStorage<Note[]>('notes', []);
```

## 6. 動作仕様

### 6.1 アプリ起動時の動作
1. LocalStorageから`'notes'`キーのデータを読み込む
2. データが存在する場合は、そのデータをメモ一覧に表示
3. データが存在しない場合は、空の配列で初期化
4. メモは更新日時の新しい順にソートして表示

### 6.2 メモ作成時の動作
1. 「+ 新規メモ」ボタンがクリックされる
2. 新しい`Note`オブジェクトが作成される（IDは現在のタイムスタンプ）
3. メモ配列の先頭に追加される
4. 新しく作成されたメモが自動的に選択される
5. LocalStorageに保存される

### 6.3 メモ編集時の動作
1. メモ一覧からメモが選択される
2. エディターパネルにメモの内容が表示される
3. ユーザーがタイトルまたは本文を編集する
4. 入力のたびに`updateNote`が呼ばれる
5. `updatedAt`が現在時刻に更新される
6. LocalStorageに即座に保存される
7. メモ一覧が自動的に再ソートされる（更新日時の新しい順）

### 6.4 メモ削除時の動作
1. メモ項目の「×」ボタンがクリックされる
2. クリックイベントの伝播が停止される（メモ選択が発生しない）
3. メモ配列から該当メモが削除される
4. 削除されたメモが選択されていた場合、選択状態が解除される
5. LocalStorageに保存される

### 6.5 データ永続化の動作
- **保存タイミング**: メモの作成・更新・削除が行われるたびに即座に保存
- **読み込みタイミング**: アプリ起動時のみ
- **エラー処理**: LocalStorageへのアクセスに失敗した場合、コンソールにエラーを出力するが、アプリは動作を継続する

## 7. 制約事項

### 7.1 ブラウザ依存
- LocalStorageを使用しているため、ブラウザのプライベートモードではデータが保持されない
- LocalStorageの容量制限（通常5-10MB）に依存する

### 7.2 データ形式
- メモIDはタイムスタンプベースのため、同時に複数のメモを作成した場合、IDの衝突の可能性がある（実用上は問題ない）

### 7.3 パフォーマンス
- メモ数が非常に多い場合（数千件以上）、パフォーマンスの低下が予想される
- 現在の実装では、すべてのメモを一度に表示する
